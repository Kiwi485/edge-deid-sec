

# 一、研究方法與步驟

### （一）舌部影像 ROI 前處理流程與效能量測方法

本研究第一年聚焦於系統層級之可行性驗證，評估舌部影像前處理流程於一般運算環境下之即時性與資源需求。實驗以筆電內建攝影機即時擷取影像串流，連續處理 **100 張影像幀**，並於每一影像幀中執行代表性舌部 ROI（Region of Interest）裁切與基本影像處理操作（影像裁切與灰階轉換），以模擬邊緣端舌部影像前處理之最小可行流程。

系統於每一影像幀處理過程中，同步量測單幀處理延遲（latency），並即時記錄 CPU 使用率與記憶體使用量。所有量測結果皆以結構化格式（CSV 檔案）進行保存，作為系統效能分析與後續低資源環境測試之比較基準。

---

# 二、初步實驗結果與數據

### （二）系統效能評估結果（Latency 與資源使用）

根據初步實驗結果，在筆電環境下執行代表性舌部 ROI 前處理流程時，單一影像幀之處理延遲介於 **0.04 ms 至 1.27 ms** 之間。其中，除第一影像幀因初始化造成較高延遲外，其餘多數影像幀之處理延遲集中於 **約 0.04–0.08 ms**，顯示該流程具備穩定且極低延遲之特性。

在運算資源使用方面，CPU 使用率多數時間維持於 **約 5%–35%** 區間，僅於少數影像幀出現瞬時較高波動；整體未觀察到持續性高負載情形。記憶體使用量約介於 **88 MB 至 95 MB** 之間，於整體實驗過程中呈現穩定狀態，未出現明顯累積或異常成長現象。

上述結果顯示，本研究所實作之舌部影像 ROI 前處理流程，在即時影像串流條件下可維持低延遲、低運算負載與穩定運行之系統特性。

> （本研究已完整保留每一影像幀之處理延遲、CPU 使用率與記憶體使用量 log，作為系統可行性驗證之佐證資料。）

# 三、邊緣運算可行性說明

### （三）邊緣設備部署可行性分析

由初步效能量測結果可知，所提出之舌部影像 ROI 前處理流程於一般筆電運算環境下即可即時穩定執行，其處理延遲與資源需求均顯著低於一般邊緣運算設備之可負荷範圍。此結果顯示，本研究之系統架構具備部署於低資源環境（如限制 CPU 與記憶體配置之虛擬機，或樹莓派等邊緣設備）之可行性。

後續研究將於低資源虛擬機環境中執行相同之處理流程與效能量測，進一步比較不同運算資源條件下之系統表現，作為實體邊緣設備部署前之重要驗證依據。

---

# 四、可放在「預期成果（第一年）」的精簡版（可選）

> 本研究第一年已完成舌部影像 ROI 前處理流程之原型實作，並取得具體效能量測數據。初步結果顯示，該流程於即時影像串流條件下，單幀處理延遲多數低於 0.1 ms，CPU 使用率約為 5%–35%，記憶體使用量約為 88–95 MB，顯示其具備低延遲與低資源需求之特性，適合作為後續智慧舌診系統之邊緣端前處理模組。
